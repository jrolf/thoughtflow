# Contributing

ThoughtFlow welcomes contributions — bug reports, provider additions, documentation fixes, and new primitives that earn their place. The project values a small API surface, explicit state, zero dependencies, and deterministic testing; contributions are reviewed against those principles. Start with an issue for anything substantial, keep changes focused, and make sure unit tests pass offline.

The full contribution guide — development setup, workflow, commit conventions, and review process — lives in the repository:

**[CONTRIBUTING.md on GitHub](https://github.com/jrolf/thoughtflow/blob/main/CONTRIBUTING.md)**

For step-by-step maintainer guides (environment setup, testing, adding providers, releasing), see the [developer/ directory](https://github.com/jrolf/thoughtflow/tree/main/developer).
